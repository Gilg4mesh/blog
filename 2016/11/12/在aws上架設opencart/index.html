在AWS上架設OpenCart

發表於 2016-11-12   |   分類於 網頁 ， 雲端服務   |  

在 AWS 上利用 EC2 及 CloudFront 架設 OpenCart 系統

環境介紹

這次因為剛好辦了 amazon 的帳號有一年試用時間所以以這裡的環境作為示範

不會用英文介面的話可以考慮使用 Azure 或是中華電信的雲端服務

下圖是有用到的兩個服務

倉儲空間不夠的話可以考慮再串接 S3 的服務

Opencart 安裝

創建EC2虛擬機

生成一組 key 讓 opencart 安裝時能直接用 (可以直接創虛擬機讓他要求生成 key)

然後之後 SSH 也要用所以先把 key 載下來

使用 awsmarketplace 安裝 opencart

此處使用 OpenCart powered by Bitnami

版本為 2.3.0.2-1-r27 on Ubuntu 14.04.3

連結 : https://aws.amazon.com/marketplace/pp/B00YW7P43K

  1. 登入之後點選 Continue

  1. 選擇要使用的虛擬機規格(我只有免費試用所以用 t2.micro)
  2. 然後最下面選擇你一開始創的 key
  3. 點選 Launch with 1-click

  1. 然後就幫你裝好了 ! (好啦其實虛擬機那邊要等他跑一下 :p)

附錄. opencart 管理者帳密

用這個套件安裝完 opencart 後他會先自動把你的管理員帳密設定好

帳號是 user，密碼比較麻煩要自己去看

在裝了 opencart 的虛擬機右鍵 -> Instance Settings -> Get System Log

取得預設密碼把它存起來之後再自己去修改

SSH 連線到虛擬機

使用 PuTTYgen 轉換您的私有金鑰

PuTTY 本身不支援 Amazon EC2 生成的私有金鑰格式 (.pem)

PuTTY 有一個名為 PuTTYgen 的工具，可將金鑰轉換成所需的 PuTTY 格式 (.ppk)

必須將私有金鑰轉換為此格式 (.ppk)，然後才能嘗試使用 PuTTY 連接到實例

  1. 在 Type of key to generate (要生成的金鑰類型) 下，選擇 SSH-2 RSA
  2. 按下”Load”選擇顯示所有類型 (在預設情況下，PuTTYgen 僅顯示副檔名為 .ppk 的文件)
  3. 選擇啟動實例時指定的金鑰 .pem 文件，按下 Open，按下 OK 關閉確認
  4. 按下 Save private key 後會顯示一條關於在沒有口令的情況下保存金鑰的警告，按下 Yes

使用 PuTTY 連線主機

  1. 在 Host Name (主機名稱) 框中，輸入 user_name@public_dns_name
  2. 在 Connection type 選擇 SSH，Port 為 22
  3. 在 Category 中，展開 Connection，再展開 SSH，然後選擇 Auth，Browse 剛剛生成的 .ppk
  4. 回到 Session 把設定存下來以便下次用
  5. Open 開啟連線 (警告詢問您是否信任您要連接到的主機可直接按下 YES)

使用 FileZilla 上傳/下載檔案

  1. 編輯->設置->連接->SFTP->加入金鑰檔案（用一開始的.pem即可）->確認
  2. 檔案->站台管理員->新增站台
  3. 主機輸入 EC2 的 Public DNS，協定選 SFTP，登入形式選一般，使用者為 ubuntu
  4. 連線 (這次安裝的 opencart 頁面目錄在 /opt/bitnami/apps/opencart/htdocs)

附錄. opencart 中文化

使用 ntcart 製作的繁體包 : https://www.opencart.com/index.php?route=marketplace/extension/info&extension_id=27435

下載後把 language 裡的語言包用 FileZilla 放到對應的目錄下

  1. 到管理員介面-> System -> Localisation -> Language -> Add New ->必填項目都填 zh-TW -> Save -> 新增完成 !

  1. 再到 System -> Settings ->把前 Language 都改成 zh-TW -> Save -> Done !

使用 CDN

  1. 開通 CloudFront 服務
  2. 沒有用過的話首先點 Create Distribution
  3. 在 web 那邊點 Get Started
  4. Origin Domain Name 填上 EC2 的 Public DNS

  1. 可以選擇添加CNAME (只需要輸入要使用的二級網址 cdn.DomainName 即可，此處設置後需要步驟7)

  1. 按下建立按鈕，先去看集動畫再回來，Status 應該就從 InProgress 變成 Depolyed 了

  1. 有 CNAME 的話要到自己的網址注冊商處添加一條該 CNAME 的 DNS 記錄指到 CloudFront 的 Domain name

用 https://developers.google.com/speed/pagespeed/insights/ 測試分配到的 xxx.cloudfront.net 確實有比較快

名詞解釋

  • CDN

全稱是 Content Delivery Network，即內容分髮網絡。其目的是通過在現有的Internet中增加一層新的 CACHE(緩存)層，將網站的內容發布到最接近用戶的網絡」邊緣「的節點，使用戶可以就近取得所需的內容，提高用戶訪問網站的響應速度。從技術上全面解決由於網絡帶寬小、用戶訪問量大、網點分布不均等原因，提高用戶訪問網站的響應速度。

  • CNAME

全稱是 Canonical Name，別名指向，將 www.domain.com 或 cdn.domain.com 等來源網域對應至目的地網域的 DNS 功能。在這種情況下，來源網域是使用者的自訂網域和子網域 (www 或 cdn 子網域一律是必要的項目)。目的地網域是 CDN 端點。
