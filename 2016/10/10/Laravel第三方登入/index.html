Laravel第三方登入

發表於 2016-10-10   |   分類於 網頁   |  

在 cloud9 上實作 php Laravel 的 fb 及 gmail 第三方登入

需求

Web Server & PHP

Apache or Nginx and PHP 5.5.9 or higher

GIT & Composer

sudo apt-get update

sudo apt-get install git

然後利用 Composer package manager 安裝 Laravel 及其他 package

Setting up the project

創立新project

composer create-project laravel/laravel social-login

執行完後會生成一個叫 social-login 的資料夾

接下來要安裝 Socialite 的 package

composer require laravel/socialite

然後打開 config/app.php 在 providers 加上

在 aliases 加上

畫面

設計方面用到了 Twitter Bootstrap, Font Awesome 及 Bootstrap Social Buttons

Master Page

在 resources/views 內新增一個 layouts 資料夾

然後新增一個 master.blade.php 檔案在裡面貼上

主要頁面

在 resources/views 內新增 home.blade.php 然後貼上

用 @extends 來繼承 Master Page 然後用 @section 來改變內容

最後要用 @stop 結尾

改掉 app/Http/routes.php 中的路徑讓他指向 home.blade.php

這時候執行網頁去跟目錄就可以看見簡單的首頁畫面

登入頁面

同樣在 resources/views 新增 login.blade.php 貼上

在改變 Router 前要先生成一個 Controller 來控制登入前/後的畫面

php artisan make:controller LoginController

然後會發現 app/Http/Controllers 裡面多了一個 LoginController.php

在裡面新增一個 showLoginPage 的 function

然後在剛剛的 app/Http/routes.php 內新增 login Route 並讓他指向 LoginController 的 showLoginPage

接下來在 master.blade.php 加點東西把 navbar 上的 Sign In 指向 /login

此時按下網頁的 Sign In 就會到 /login 下的登入畫面了

註冊第三方登入的網頁應用

FB 跟 Google 的畫面因為常會變動所以就略過

總之都會拿到 App Id 跟 App Secret

在 config/services.php 內新增需要的 key

然後在 .env 內新增

認證及登入

首先處理資料庫的部分，Laravel 預設在 database/migrations 內有個類似叫
2014_10_12_000000_create_users_table.php 的檔案處理資料庫遷移

在裡面貼上以下

為了第三方登入我們要再新增另一個遷移檔

php artisan make:migration create_social_login_profiles_table --create=social_login_profiles

執行完後會在 database/migrations 內多一個類似 2016_06_03_204539_create_social_login_profiles_table.php 的檔案

在裡面貼上以下

登入所使用的資料庫並新增一個 social_login 的資料庫並更新我們的 .env

然後執行剛剛的 migration 檔並新增 tables

php artisan migrate
